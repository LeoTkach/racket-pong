# Быстрое решение проблемы CORS

## Текущая ситуация

Бакет Firebase Storage еще не создан. Firebase создает бакет автоматически при первом использовании Storage.

## Решение (2 шага)

### Шаг 1: Создайте бакет (загрузите файл один раз)

Попробуйте загрузить файл через ваше приложение один раз. Это создаст бакет автоматически.

**ИЛИ** создайте бакет вручную через Firebase Console:
1. Откройте https://console.firebase.google.com/
2. Выберите проект `racket-pong`
3. Перейдите в **Storage**
4. Нажмите **Get started** (если Storage еще не включен)
5. Следуйте инструкциям для создания бакета

### Шаг 2: Примените CORS конфигурацию

После создания бакета, запустите:

```bash
./setup-firebase-cors.sh
```

Или вручную:

```bash
export PATH=/opt/homebrew/share/google-cloud-sdk/bin:"$PATH"
gsutil cors set cors.json gs://racket-pong.firebasestorage.app
```

## Альтернативное решение

Если вы не можете создать бакет сейчас, вы можете временно использовать локальное хранилище файлов (как было раньше) до тех пор, пока не настроите Firebase Storage.

## После настройки

После применения CORS конфигурации, ошибка CORS должна исчезнуть и загрузка файлов будет работать нормально.
